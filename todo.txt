KONTEKS PROJECT:
Project ini adalah aplikasi Habit Tracker multi-role (Student, Mentor, Admin) dengan stack:
- Laravel 12
- Inertia.js + Vue 3
- Filament v3 (Mentor/Admin panel)
- Spatie Permission (role management)

Fitur existing memiliki area sensitif:
- Role & Middleware redirect
- Habit ownership & pivot logic
- Habit toggle system (HabitLog)
- Habit assignment (duplikasi data oleh Mentor)

TUGAS:
Tambahkan fitur baru bernama **Presensi / Kehadiran Student** dengan alur berikut:

FLOW YANG DIINGINKAN:
1. Default status kehadiran Student = TIDAK HADIR
2. Student dapat menandai dirinya sebagai "Hadir" (check-in)
3. Status tersebut BELUM VALID sampai Mentor melakukan konfirmasi
4. Mentor melihat daftar pengajuan kehadiran dari student dalam classroom
5. Mentor dapat:
   - Accept (menjadi valid)
   - Reject (kembali ke tidak hadir / ditolak)
6. Data yang sudah di-accept akan menjadi data absensi final
