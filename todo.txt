### BAGIAN 1: DIAGNOSA & FIX ERROR 403 (Priority)
Saya ingin kamu menelusuri kode saya untuk menemukan penyebab logic yang menyebabkan 403 ini. Jangan hanya menebak, tapi audit kode berikut:

1. **Audit Route & Controller:**
   - Cari definisi route untuk URL `/habits/{habit}/toggle/{date}`.
   - Periksa Controller method yang menangani route tersebut.
   - Cek apakah ada Authorize Gate atau Policy (contoh: `$this->authorize('update', $habit)`). Jika ada, pastikan logic di `HabitPolicy` tidak salah membandingkan `user_id` (misalnya masalah tipe data integer vs string).

2. **Audit Middleware & Config (Crucial for Production):**
   - Cek file `app/Http/Middleware/TrustProxies.php` atau `bootstrap/app.php` (untuk Laravel 11). Pastikan proxies dikonfigurasi dengan benar (`protected $proxies = '*';`) agar aplikasi mengenali request HTTPS dari proxy/load balancer.
   - Cek konfigurasi `config/session.php` dan logic `.env`. Pastikan `SESSION_DOMAIN` dan `SECURE_COOKIE` dikonfigurasi agar tidak menolak cookie di production (masalah CSRF).

3. **Berikan Solusi Code:**
   - Jika ada kesalahan logic di Policy, perbaiki.
   - Jika konfigurasi Proxy kurang, tambahkan kodenya.
