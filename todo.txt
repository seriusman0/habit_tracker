FINAL PRODUCTION SECURITY & STABILITY AUDIT

TUJUAN UTAMA:
Pastikan project benar-benar siap deploy ke server hosting.
Tidak ada celah keamanan tersisa.
Tidak ada konfigurasi development yang tertinggal.
Semua error ditangani dengan aman dan user-friendly (tidak teknis).

Jangan menambahkan fitur baru.
Fokus hanya audit, hardening, dan kesiapan production.

==================================================
1. NONAKTIFKAN SEMUA MODE DEVELOPMENT
==================================================
Pastikan:
- APP_ENV=production
- APP_DEBUG=false
- Tidak ada dd(), dump(), console.log() tersisa.
- Tidak ada route testing/debug aktif.
- Tidak ada file percobaan di public folder.

Cache ulang:
- config
- route
- view

==================================================
2. HARDENING KEAMANAN WAJIB
==================================================

A. AUTHORIZATION
- Semua route memiliki middleware auth.
- Route admin hanya bisa diakses admin.
- Route mentor hanya bisa diakses mentor.
- Route student hanya bisa diakses student.
- Tidak ada IDOR (akses data via manipulasi ID URL).

B. VALIDASI INPUT
- Semua form menggunakan FormRequest atau validator.
- Tidak ada input tanpa validasi.
- Sanitasi input string.
- Cegah mass assignment (gunakan $fillable).

C. CSRF & SESSION
- CSRF aktif di semua form.
- Session httpOnly dan secure (jika HTTPS).
- Regenerate session setelah login.

D. PASSWORD
- Gunakan hashing bcrypt/argon.
- Tidak ada password plain text di database.
- Rate limit login aktif.

==================================================
3. ERROR HANDLING (WAJIB USER-FRIENDLY)
==================================================

Pastikan:

A. APP_DEBUG=false.
B. Buat halaman custom:
   - 403 (Akses tidak diizinkan)
   - 404 (Halaman tidak ditemukan)
   - 500 (Terjadi kesalahan sistem)

Ketentuan:
- Tidak menampilkan stack trace.
- Tidak menampilkan path file.
- Tidak menampilkan query database.
- Gunakan bahasa yang mudah dipahami user non-teknis.

Contoh gaya pesan:
"Terjadi gangguan pada sistem. Silakan coba beberapa saat lagi."

C. Logging tetap aktif di storage/logs untuk admin, tapi tidak terlihat oleh user.

==================================================
4. DATABASE & DATA SAFETY
==================================================

- Semua foreign key aktif.
- Tidak ada orphan record.
- Index tersedia untuk kolom relasi penting.
- Migration sudah final.
- Tidak ada kolom testing yang tidak terpakai.

Pastikan:
- Relasi classroom, mentor, student konsisten.
- Reporting admin dan mentor memakai service layer yang sama.
- Reset daily & weekly berjalan sesuai aturan.

==================================================
5. PERFORMA DASAR
==================================================

- Tidak ada N+1 query.
- Gunakan eager loading.
- Gunakan pagination untuk data besar.
- Tidak ada query berat di blade/view.
- Tidak ada loop perhitungan kompleks di frontend.

==================================================
6. FILE & PERMISSION CHECK
==================================================

- storage/ dapat ditulis.
- bootstrap/cache/ dapat ditulis.
- Tidak ada file .env dapat diakses publik.
- Nonaktifkan directory listing di hosting.

==================================================
7. FINAL ROUTE & FEATURE AUDIT
==================================================

Periksa manual:

ADMIN:
- Classroom attach student sinkron.
- Reporting tampil benar.

MENTOR:
- My-students konsisten dengan admin.
- Tidak bisa akses student luar class.

STUDENT:
- Daily reset per tanggal.
- Weekly reset hari Minggu.
- Friends hanya class yang sama.

==================================================
8. CLEANUP DEVELOPMENT ARTIFACT
==================================================

Hapus:
- File backup lama.
- File .sql di public.
- Dokumentasi internal di root.
- Script testing.
- Seeder yang tidak diperlukan di production.

==================================================
9. KONFIRMASI AKHIR SEBELUM DEPLOY
==================================================

Pastikan:
- Tidak ada error di log terbaru.
- Tidak ada warning di browser console.
- Semua role terisolasi dengan benar.
- Error page tampil ramah user.
- Sistem tetap stabil jika data kosong.

OUTPUT YANG DIHARAPKAN:
1. Daftar potensi celah keamanan (jika ada).
2. Daftar perbaikan yang dilakukan.
3. Konfirmasi bahwa sistem aman untuk deploy manual ke hosting.
