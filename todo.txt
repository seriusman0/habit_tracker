🎯 Konteks

Route /student/friends sudah dibuat sebelumnya.

Saat ini halaman tersebut masih kosong / belum memiliki logic dan tampilan.

Tugas: Mengisi halaman tersebut dengan fitur ringkasan pencapaian teman sekelas.

🧠 Prompt untuk AI

Implementasikan fitur pada halaman /student/friends yang sudah tersedia dengan ketentuan berikut:

1️⃣ Akses dan Scope Data

Hanya student yang login dapat mengakses halaman ini.

Student hanya dapat melihat:

Student lain yang berada dalam class yang sama.

Jangan tampilkan student dari class lain.

Jika student belum tergabung dalam class → tampilkan empty state yang jelas dan tidak error.

2️⃣ Data yang Harus Ditampilkan

Untuk setiap teman dalam class yang sama, tampilkan:

Nama student

Ringkasan pencapaian minggu berjalan (Senin–Minggu)

Ringkasan terdiri dari:

Total habit daily yang diselesaikan minggu ini

Total habit daily yang seharusnya dikerjakan minggu ini

Total habit weekly yang diselesaikan minggu ini

Total habit weekly yang seharusnya dikerjakan minggu ini

Persentase completion keseluruhan minggu berjalan

Contoh struktur hasil akhir yang di-render:

Nama: Budi
Daily: 18 / 21
Weekly: 2 / 3
Progress: 83%
3️⃣ Aturan Perhitungan Mingguan

Minggu dimulai hari Senin 00:00:00

Berakhir hari Minggu 23:59:59

Gunakan timezone sistem.

Logika:

Habit daily dihitung berdasarkan record per tanggal dalam range minggu berjalan.

Habit weekly dihitung satu kali dalam minggu tersebut.

Jika habit weekly sudah diisi sekali, maka dianggap complete untuk minggu itu.

Pastikan tidak menghitung data di luar minggu berjalan.

4️⃣ Struktur Backend

Jangan letakkan logic di view.

Pisahkan menjadi:

FriendController

WeeklySummaryService

Query menggunakan eager loading

Gunakan aggregation query untuk menghitung summary

Hindari N+1 query.

5️⃣ UI/UX Requirement

Tampilan harus:

Responsif (mobile-first)

Menggunakan card/grid layout

Clean dan konsisten dengan dashboard student

Tidak terlalu padat

Struktur card sederhana:

[Avatar / Initial]
Nama Student

Daily: 18 / 21
Weekly: 2 / 3
Progress Bar 83%

Ketentuan UI:

Gunakan progress bar visual

Gunakan warna netral

Ikon konsisten dengan design system sidebar

Hindari animasi berlebihan

Pastikan tidak error ketika data kosong

6️⃣ Edge Case Handling

Tangani kondisi berikut:

Student belum punya class → tampilkan pesan informatif.

Student tidak punya teman → tampilkan empty state:
“Belum ada teman di kelas ini.”

Student belum punya habit → tampilkan 0%.

Hindari null reference dan undefined variable.

Jika avatar kosong → tampilkan inisial nama.

7️⃣ Keamanan

Validasi bahwa student hanya dapat melihat data class miliknya.

Jangan expose:

Email

Role

Informasi sensitif lainnya

8️⃣ Performa

Jika 1 class memiliki banyak student:

Gunakan pagination (maksimal 20 per halaman).

Tambahkan optional search sederhana.

Pastikan query tetap efisien.

9️⃣ Validasi Final

Sebelum menyelesaikan implementasi, pastikan:

Tidak ada error undefined variable.

Tidak ada N+1 query.

Perhitungan minggu sesuai standar Senin–Minggu.

UI tidak rusak di layar mobile.

Data tidak bocor ke class lain.

Halaman /student/friends benar-benar terisi dan menampilkan data dinamis.

Hasil akhir: halaman /student/friends yang sebelumnya kosong kini menampilkan daftar teman sekelas beserta ringkasan pencapaian minggu berjalan secara rapi, ringan, dan dapat dianalisis dengan cepat oleh student.
