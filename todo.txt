Anda adalah Senior Laravel Architect dengan pengalaman mendalam menggunakan Sharp CMS (code16/sharp) versi terbaru (v8/v9).

KONTEKS PROJECT:
- Framework: Laravel (versi modern)
- CMS: Sharp (code16/sharp)
- Stack UI Sharp: Inertia + Tailwind (default Sharp)
- Sharp sudah berhasil diakses di URL /sharp
- Login Sharp sudah berfungsi
- Sidebar Sharp masih default dan sangat terbatas

TUJUAN UTAMA:
Saya ingin Sharp menjadi CMS/Admin Panel utama dengan kontrol penuh terhadap:
1. Manajemen User
2. Manajemen Role
3. Relasi User ↔ Role
4. Akses menu Sharp berbasis role
5. CRUD berjalan stabil dan tanpa error

TUGAS ANDA (WAJIB DIKERJAKAN SEMUA):

========================
A. DESAIN STRUKTUR DATA
========================
1. Tentukan struktur tabel:
   - users (asumsi sudah ada)
   - roles
   - relasi user_role (pivot atau foreign key)
2. Buat migration yang BENAR dan aman.
3. Tentukan relasi Eloquent (User ↔ Role).

========================
B. SHARP ENTITIES
========================
Buat Sharp Entity lengkap untuk:
1. User
   - List
   - Form
   - Show (opsional tapi direkomendasikan)
2. Role
   - List
   - Form

Pastikan:
- Menggunakan Sharp class versi terbaru
- Tidak ada class deprecated
- Signature method sesuai SharpForm / SharpEntityList
- Semua `use` namespace BENAR

========================
C. CRUD REQUIREMENTS
========================
USER:
- List user (nama, email, role)
- Create user
- Update user
- Delete user
- Assign role ke user
- Validasi form (email unique, required fields)

ROLE:
- List role
- Create role
- Update role
- Delete role (dengan proteksi jika masih dipakai user)

========================
D. SIDEBAR SHARP
========================
1. Tambahkan menu sidebar:
   - Users
   - Roles
2. Kelompokkan menu secara rapi (misalnya: "Access Management").
3. Pastikan menu muncul dan bisa diakses.

========================
E. AUTHORIZATION (WAJIB)
========================
1. Hanya user dengan role tertentu (misalnya `admin`) yang bisa:
   - Mengakses Sharp
   - Mengelola user & role
2. Gunakan Sharp authorization / Laravel policy yang benar.
3. Jelaskan titik kontrol aksesnya.

========================
F. FILE & STRUKTUR
========================
Tampilkan struktur folder Sharp yang IDEAL:
- app/Sharp/
  - Users/
  - Roles/
  - SharpServiceProvider.php
  - SharpMenu.php (jika digunakan)

========================
G. KODE FINAL
========================
Untuk setiap komponen:
- Migration
- Model
- Sharp Entity List
- Sharp Form
- Sharp Configuration / Menu

WAJIB:
- Kode siap copy-paste
- Tidak ada placeholder ambigu
- Tidak ada pseudo-code
- Tidak ada class deprecated

========================
H. STABILITAS & VERIFIKASI
========================
1. Sebutkan perintah terminal WAJIB setelah implementasi:
   - migrate
   - clear cache
   - autoload
2. Daftar error umum Sharp terkait Users/Roles dan solusinya.
3. Checklist verifikasi akhir (login → sidebar → CRUD).

========================
BATASAN KERAS:
- Jangan gunakan FormFieldsContainer

OUTPUT:
