ROLE-BASED FEATURE REPLICATION (STUDENT TO MENTOR)

ROLE:
Anda adalah Senior Full-stack Developer yang sangat teliti dalam manajemen Role & Permission (Spatie) dan arsitektur Laravel/React.

CONTEXT:

Existing Feature: Saya sudah memiliki fitur "Kebiasaanku" di /student (Role: Student). Fitur ini memiliki UI khusus (seperti screenshot yang saya berikan) dan logika backend untuk mencatat habit.

Goal: Saya ingin role "Mentor" memiliki fitur yang identik di /mentor/my-habits. Mentor harus bisa setup habit mereka sendiri dan memiliki UI yang sama persis dengan yang dimiliki Student.

Tech Stack: Laravel, React (Inertia.js), dan Spatie Permission.

STEP 1: ARCHITECTURAL AUDIT (MANDATORY)
Sebelum menulis kode, pelajari bagaimana fitur ini bekerja agar tidak terjadi duplikasi kode yang sia-sia (DRY Principle). Mohon minta saya mengirimkan:

routes/web.php (Bagian middleware student dan mentor).

Controller yang menangani fitur "Kebiasaanku" di sisi Student.

Komponen React utama untuk halaman /student tersebut.

File User.php (Model) untuk melihat relasi habit.

STEP 2: ANALYSIS & STRATEGY
Setelah membaca file tersebut, berikan analisis Anda:

Reusability: Apakah kita bisa menggunakan Controller yang sama dengan logic berbeda, atau harus membuat MentorHabitController baru?

UI Consistency: Bagaimana cara memanggil komponen UI "Kebiasaanku" di dalam Layout Mentor tanpa merusak CSS/styling dashboard mentor?

Spatie Logic: Pastikan permission diatur agar Mentor hanya bisa melihat habit miliknya sendiri, bukan milik student.

STEP 3: EXECUTION PLAN
Berikan panduan langkah demi langkah untuk:

Penambahan Route di grup middleware 'mentor'.

Penyesuaian Backend (Controller & Model).

Implementasi Frontend di halaman baru /mentor/my-habits.

TUNGGU INSTRUKSI: Jangan berikan kode sekarang. Katakan: "Saya siap membantu menduplikasi fitur 'Kebiasaanku' untuk Mentor. Agar tidak terjadi error role-conflict, tolong lampirkan isi file web.php dan Controller fitur habit yang sudah ada."
